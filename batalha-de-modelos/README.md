# 🚀 Batalha de Modelos & Engenharia de Prompt (XML)

Projeto acadêmico voltado para análise comparativa entre diferentes modelos de Inteligência Artificial utilizando Engenharia de Prompt em XML para geração de uma página HTML Single Page com CSS integrado.

---

## 📚 Sobre o Projeto

O objetivo desta atividade foi avaliar a capacidade de diferentes modelos de IA em interpretar e executar um prompt estruturado em XML, gerando uma aplicação web responsiva seguindo requisitos específicos.

### Modelos analisados

* ChatGPT
* Gemini
* DeepSeek
* Qwen
* Grok
* Maritaca
* Claude

---

# 🎯 Objetivo da Tarefa

Criar uma página HTML5 única contendo:

* CSS3 interno
* Layout responsivo
* Design minimalista
* Menu hambúrguer funcional
* Seções:

  * Perfil
  * Atividades
  * Configurações
* Rodapé com contatos simulados

---

# 🧩 Prompt Estruturado Utilizado

```xml
<tarefa>
    <objetivo>
        Criar uma página HTML5 única com CSS3 interno (single page).
    </objetivo>

    <tema>
        Área de Atividades Academicas
    </tema>

    <diretrizes_design>
        <layout>Responsivo e minimalista.</layout>
        <paleta_cores>Branco e Azul</paleta_cores>
        <tipografia>
            Sans-serif para títulos, Serif para corpo.
        </tipografia>
    </diretrizes_design>

    <obrigatoriedades_tecnicas>
        <item>
            Menu de navegação funcional em hamburguer(âncoras).
        </item>

        <item>
            Seção de Perfil, Atividades e Configurações.
        </item>

        <item>
            Rodapé com informações de contato simuladas.
        </item>
    </obrigatoriedades_tecnicas>

    <metrica_obrigatoria>
        Ao final da resposta, informe uma estimativa
        de quantos tokens foram gerados para este código.
    </metrica_obrigatoria>
</tarefa>
```

# 📌 Conclusão

A experiência demonstrou como diferentes modelos de IA interpretam estruturas formais de prompt de maneiras distintas, mesmo recebendo exatamente o mesmo contexto.

O uso de XML ajudou a:

* Melhorar a organização do prompt
* Reduzir ambiguidades
* Facilitar validações estruturais
* Comparar precisão entre modelos

Também foi possível observar diferenças claras em:

* Criatividade
* Fidelidade técnica
* Quantidade de tokens
* Organização do código
* Aderência aos requisitos
